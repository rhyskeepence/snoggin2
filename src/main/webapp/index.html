Or, choose your own adventure

Start with this URL:

chart.html?

add some fields

fields=

Fields are in the format 

[system-name]:[metric]
(separated by commas)

[system-name] being something like 

production-activemq
or 
production-knitware

[metric] being something like 

jvmThreadCount

Then choose the number of days to display, like this: 

&days=90
If you leave this off, the default will show you a chart of the last 7 days

You can optionally choose how to aggregate: 

&aggregate=sum

  1. sum will add up all the values per day
  2. average will average all the values per day (recommended for things such as thread counts, memory, CPU, etc)
  3. errors will show the amount of downtime, in minutes, per day
  4. dailythroughput is special - it's used for activemq's ThroughputPerSecond metric, which will be totalled up to give throughput per day
  5. If you don't include the aggregate parameter, you'll see a high resolution chart of the metric
